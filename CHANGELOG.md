# Changelog

## [1.0.2] - 2024-01-XX

### 🔧 增强端口范围管理

#### ✨ 新增功能
- **重新添加endPort属性**: 支持配置端口范围的结束端口
- **灵活的端口范围管理**: 支持自动突破配置范围寻找可用端口
- **智能端口分配**: 优先使用配置范围，不足时自动扩展

#### 🔄 端口分配策略
1. **优先级1**: 使用配置的端口池中的端口
2. **优先级2**: 使用配置的端口范围（startPort到endPort）
3. **优先级3**: 如果指定范围不足，自动突破范围寻找可用端口
   - 向上突破：endPort+1 到 endPort+100
   - 向下突破：startPort-1 到 startPort-100
4. **冲突处理**: 如果端口被占用，自动寻找下一个可用端口

#### 📝 配置验证
- **端口范围验证**: endPort必须大于startPort
- **默认值处理**: 如果未指定endPort，使用startPort + 100
- **错误处理**: 提供详细的错误信息和日志记录

#### 🎯 优势
- **更灵活的配置**: 用户可以精确控制端口范围
- **自动容错**: 配置范围不足时自动扩展，确保服务可用
- **更好的日志**: 提供详细的端口分配过程日志

#### 📚 文档更新
- 更新了README中的代理配置说明
- 添加了端口分配策略的详细说明
- 更新了测试用例，包含endPort属性的测试
